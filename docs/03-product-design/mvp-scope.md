# MVP Scope

## Objective

The goal of the Minimum Viable Product (MVP) is to validate the DS-Forge concept quickly by launching a simplified version of the platform.

The MVP will focus on core functionality required for users to practice data problems.

## Core MVP Features

### User Authentication
Users can register, login, and manage their profile.

### Problem Library
A collection of curated SQL and Python data problems.

### Dataset Viewer
Users can view tables and dataset schema.

### Code Editor
In-browser editor for writing SQL or Python code.

### Code Execution
User code is executed in a sandbox environment.

### Evaluation Engine
System verifies user output against expected results.

### AI Assistance
AI provides hints and explanations for problems.

### Leaderboard
Users are ranked based on problem performance.

## Features Excluded From MVP

The following features will be introduced later:

- ML challenge competitions
- collaborative coding
- community discussions
- enterprise hiring tools

## MVP Success Criteria

The MVP will be considered successful if:

- users actively solve problems
- users use AI hints
- users return to the platform

## Expected MVP Timeline

Development phases:

Phase 1: System architecture and backend  
Phase 2: frontend interface  
Phase 3: AI integration  
Phase 4: testing and launch
